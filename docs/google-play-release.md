# Google Play Release Preparation

Last updated: 2026-06-18

This document prepares the Android release of `ZHAO's Family` for Google Play
internal testing. The app is positioned as an internal employee tool for ZHAO
teams.

## Release Scope

- App name: `ZHAO's Family`
- Android package: `com.zhaofamily.mobile`
- First track: Google Play internal testing
- Submission method: manual upload of the `.aab` generated by EAS
- Production API: `https://api.zhaoplatforme.com/backend3`
- Privacy policy URL: `https://zhaoplatforme.com/privacy-policy`
- Account deletion URL: `https://zhaoplatforme.com/delete-account`

## Store Listing Draft

Short description:

```text
Internal employee app for ZHAO store operations, training, orders and team workflows.
```

Full description:

```text
ZHAO's Family is an internal application for ZHAO employees and store managers.
It helps authorized team members access operational workflows from one mobile app:
profile management, store information, order workflows, training materials,
training progress and recruitment requests.

The app is intended for ZHAO staff only. Access requires an approved employee
account. Store managers and administrators manage employee permissions from the
internal platform.
```

Recommended category: `Business` or `Productivity`.

Test account note for Play review:

```text
This is an internal employee app. Provide a dedicated approved test account in
Play Console before review. The account should not contain real employee data.
```

## Play Console Data Safety Draft

Final answers must be reviewed by the business or legal owner before submission.
Based on the current app behavior, disclose the following data collection:

| Data type | Collected | Shared | Required | Purpose |
| --- | --- | --- | --- | --- |
| Name | Yes | No, except service providers | Yes | Account management, app functionality |
| Email address | Yes | No, except service providers | Yes | Account management, authentication |
| User IDs | Yes | No, except service providers | Yes | Account management, security |
| Address | Yes, if provided | No, except service providers | Optional | App functionality |
| Phone number | Yes, if provided | No, except service providers | Optional | App functionality, operational contact |
| Photos | Yes, if avatar is uploaded | No, except service providers | Optional | Profile functionality |
| App interactions | Yes | No, except service providers | Yes | Training progress, orders, internal workflows |
| Device or other IDs | Yes, for push tokens | Expo push service provider | Optional | Notifications, app functionality |

Security practices:

- Data is transferred over HTTPS in production.
- Passwords are not stored in plaintext.
- Users can request account and associated data deletion from
  `https://zhaoplatforme.com/delete-account`.

Do not declare ads, public social features, location collection, payment data,
health data, contacts, calendar, SMS or installed-app inventory unless future
code adds those behaviors.

## Permissions

Android permissions expected for Play review:

- `POST_NOTIFICATIONS`: used only after user consent to receive operational push
  notifications.
- Photo library and camera access from `expo-image-picker`: used only when the
  employee chooses or captures a profile photo.
- Internet access: required for authentication and internal API calls.

`RECORD_AUDIO` was removed because the current mobile app does not use
microphone capture.

## Build And Manual Upload

From repository root:

```bash
pnpm --filter @zhao/mobile typecheck
pnpm --filter @zhao/mobile lint
pnpm --filter @zhao/mobile test
pnpm --filter @zhao/web build
pnpm --filter @zhao/mobile exec expo config --type public
pnpm --filter @zhao/mobile exec eas build -p android --profile production
```

After EAS finishes:

1. Download the generated Android App Bundle (`.aab`).
2. Open Google Play Console.
3. Create the app with package `com.zhaofamily.mobile` if it does not exist.
4. Complete App content: privacy policy, Data safety, content rating, target
   audience, ads declaration, pricing and distribution.
5. Go to Testing, then Internal testing.
6. Create a new release and upload the `.aab`.
7. Add internal testers and publish to internal testing.
8. Install from the Play internal testing link on a physical Android device.

## Internal Testing Checklist

- Install from Play internal testing link.
- Launch app without a local `.env`.
- Confirm API calls use `https://api.zhaoplatforme.com/backend3`.
- Log in with an approved test employee account.
- Register a new test account and confirm the pending approval flow.
- Reset password flow opens and submits correctly.
- Profile page opens the account deletion request URL.
- Avatar upload works from gallery and camera permission prompts are accurate.
- Push notification permission prompt appears only when needed.
- Orders, stores, training and recruitment screens load without blocking errors.
- No real employee personal data appears in screenshots or Play review notes.
